# CronWin 项目变更日志 (Change Log)

## [2026-05-28] 今日更新总结

今天针对 **CronWin 定时任务管理系统** 进行了深度的用户体验优化和高阶功能扩展。以下是今天的核心变更内容：

### 1. 📂 复杂 Cron 表达式可视化生成器 (Cron Generator)
* **核心改进**：为了方便非 Linux 运维专业人员配置定时逻辑，在任务主编辑框中集成了全新的**可视化 Cron 生成面板**。
* **支持模式**：
  * **每天定时**：秒级/分钟/小时的直观下拉框选择。
  * **每周特定天**：周一至周日的多选按钮配合特定时刻（如：周一、三、五早上 08:01 执行）。
  * **每月特定日期**：1~31 日的多选按钮。
  * **固定周期**：每隔 $N$ 分钟执行一次，或在每隔 $N$ 小时的第 $M$ 分钟执行。
  * **高级自定义**：支持手输 5 位标准的 Linux Cron 表达式。
* **反向解析支持**：打开编辑已有任务时，组件会自动对已有的常规 Cron 表达式进行特征匹配与状态反向回显。

### 2. ⚡ 消除 Windows 运行脚本时的 CMD 弹窗
* **核心改进**：解决了原先在 Windows 下执行 shell 命令由于 Node.js 默认行为导致频繁弹出黑框命令行窗口、干扰正常工作台操作的痛点。
* **技术实现**：在后台的 `exec` 选项中添加了 `windowsHide: true` 属性，使脚本命令执行完全在后台无干扰静默运行（Git Shell/CMD 下均无任何窗口跳闪）。

### 3. 📁 脚本运行工作目录 (CWD) 设定
* **核心改进**：允许针对单个任务在“选项设置”中指定独立的工作目录 `cwd`。
* **应用场景**：解决了当脚本依赖其同级目录下的其他数据、配置、模块包时必须要使用绝对路径或 `cd` 命令进行切换的麻烦。留空时默认在软件所在根目录下执行。

### 4. 🐍 任务专属局部环境变量 (支持指定特定的 Python/Bash 版本)
* **核心改进**：在任务编辑面板中增加了“局部自定义环境变量（键值对散页文本）”。
* **应用场景**：允许在各个任务中局部重写或合并 `PATH`（如通过声明 `PATH=C:\Python39;C:\Python39\Scripts;$PATH`）。这意味着用户可以在同一台主机上，让 A 脚本使用 Python 3.9，而让 B 脚本调用 Python 3.11 或特定的 Bash 编译器运行。

### 5. 📥 任务列表一键导入与导出 (JSON 格式)
* **核心改进**：在界面右上角新增了“导入任务”与“导出任务”双通道数据交互。
* **安全校验**：
  * 导出：一键拉取所有有效任务配置，滤去动态运行状态，打包为标准的带格式美化的 `.json` 静态包文件在浏览器中直接下载。
  * 导入：前台进行严格的 JSON 解析校验，对缺少核心要件（如 `name`、`script` 等）的数据进行友好拦截提示；后端接收后自动生成唯一的微秒级独立 ID、赋予初始激活状态并**动态注入底层 Cron 调度引擎实时激活**，做到导入即生效、无缝无损热迁移。
